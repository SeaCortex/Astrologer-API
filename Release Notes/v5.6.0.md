# Release Notes v5.6.0

## Highlights

-   **Relationship Score Breakdown**: The API now provides full transparency into how the compatibility score is calculated. A new `score_breakdown` field in the results details exactly which astrological rules contributed points to the final score.

## Technical Details

-   **New Field**: `score_breakdown` added to `CompatibilityScoreResponseModel`.
-   **Affected Endpoints**:
    -   `POST /api/v5/compatibility-score`
    -   `POST /api/v5/chart-data/synastry`

## Feature Description

The `score_breakdown` field is a list of objects, where each object represents a scoring rule that triggered for the couple.

### Schema

```json
{
    "rule": "string", // Identifier of the rule (e.g., 'sun_moon_conjunction', 'destiny_sign')
    "description": "string", // Human-readable description
    "points": "integer", // Points awarded
    "details": "string" // Contextual details (e.g., aspect name and orb)
}
```

### Example Usage

**Request:** `POST /api/v5/compatibility-score`

**Response:**

```json
{
    "status": "OK",
    "score": 11,
    "score_description": "Very Important",
    "is_destiny_sign": true,
    "score_breakdown": [
        {
            "rule": "sun_moon_conjunction",
            "description": "Sun-Moon conjunction (high precision)",
            "points": 11,
            "details": "Sun-Moon conjunction (orbit: 1.34°)"
        }
    ]
    // ... other fields
}
```
